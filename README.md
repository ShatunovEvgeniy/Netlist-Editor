# Hierarchical Netlist Editor & Checker System

Система автоматизированного проектирования схем, включающая редактор электрических схем, генератор нетлистов и инструмент проверки корректности. Проект разработан в соответствии со спецификацией **S0 25 spec v1**.

## Описание

Проект представляет собой набор из трех взаимосвязанных программ, предназначенных для работы с иерархическими нетлистами электрических схем. Система поддерживает кроссплатформенную работу (Windows, Linux) и предназначена для создания, редактирования и валидации описаний схем в текстовом формате.

## Компоненты системы

Система состоит из трех основных модулей:

### 1. Оболочка (Shell)
Графический лаунчер для управления компонентами системы.
- **Функции:**
  - Запуск **Редактора**.
  - Запуск **Чекера** (через диалоговое окно выбора файлов и параметров).
  - Сохранение настроек между запусками.
  - Кнопка выхода из приложения.

### 2. Редактор (Editor)
Графический редактор схем с возможностью сохранения графического вида и нетлиста.
- **Интерфейс:**
  - Строка меню и палитра инструментов.
  - Область списка блоков (создание, удаление, копирование, редактирование свойств).
  - Область графического редактирования (отображение блока как прямоугольника с внешними пинами).
- **Возможности:**
  - Работа с инстансами (примитивы и подсхемы).
  - Редактирование цепей (ломаные линии, ортогональная трассировка).
  - Различение пересечения цепей и соединения.
  - **Undo/Redo:** Неограниченная отмена действий, сохраняющаяся между запусками до момента создания файла.
  - Автосохранение изменений.
  - Блокировка файлов для записи другими программами во время работы.

### 3. Чекер (Checker)
Утилита командной строки для статического анализа нетлистов.
- **Функции:** Выявление ошибок неполноты и некорректности в файлах нетлистов.
- **Режим работы:** CLI (Command Line Interface).

## Технические требования

- **Языки программирования:** C++, Python.
- **Платформы:** Windows, Linux.
- **Компиляторы:** Общедоступные компиляторы (например, GCC, MSVC).
- **Библиотеки:** Общедоступные компоненты (список библиотек согласуется с заказчиком).
- **Формат данных:** Текстовый (ASCII), человекочитаемый.

## Установка и сборка

1. Настроить виртуальную среду:
```
python3 -m venv venv
```

2. Установить необходимые пакеты:
```
python3 -m pip install -r requirements.txt
```

3. Запустить Оболочку:
```
python3 shell.py
```

## Использование

### Работа в Редакторе
1. Создайте новый проект или откройте существующий файл схемы.
2. В области списка блоков выберите или создайте блок (корневой блок имеет имя `main`).
3. Разместите инстансы (примитивы или подсхемы) на графическом поле.
4. Соедините пины цепями (nets).
5. Сохраните проект (генерируется файл схемы и файл нетлиста).

### Запуск Чекера
1. Через **Оболочку** нажмите кнопку запуска Чекера.
2. В диалоговом окне выберите путь к файлу нетлиста.
3. Настройте параметры проверки (при необходимости).
4. Нажмите кнопку запуска для получения отчета об ошибках.

## Формат нетлиста

Нетлист представляет собой иерархическое описание схемы в текстовом виде.

- **Структура:** Файл состоит из перечисления блоков (`blocks`).
- **Корневой блок:** Всегда имеет имя `main`.
- **Блоки:** Имеют идентификатор (имя), внешние пины (`pin`) и содержат инстансы.
- **Инстансы:** Могут быть двух типов:
  - `primitive` (примитив)
  - `subcircuit` (подсхема, ссылается на другой блок)
- **Цепи (Nets):** Связывают пины внешних выходов блока и пины инстансов. К каждому пину подключено не более одной цепи.
- **Синтаксис:**
  - Строки могут быть разных типов (комментарии, пустые строки, описания элементов).
  - Элементы разделены пробелами.
  - Идентификаторы могут содержать буквы, цифры и знак подчеркивания `_`.

### Пример структуры (упрощенно)
```text
main: 
  pin a1- n1 
  pin a2- n2 
  instance b1i1: block1 
    b1i1.a1- n1 
    b1i1.a2- n2

block1: 
  pin a1- n1 
  pin a2- n2 
  instance d1: diode1 
    d1.p1- n1 
    d1.p2- n2
```

## Проверки Чекера

Чекер выявляет следующие типы ошибок:

1. **Синтаксические ошибки:** Несоответствие файла формату нетлиста.
2. **Неполнота иерархии:** В блоке указана подсхема, но определение соответствующего блока отсутствует.
3. **Зацикленность иерархии:** Цепочка вложенности блоков приводит к исходному блоку (циклическая зависимость).
4. **Некорректная подсхема:** Список пинов инстанса не соответствует списку пинов определения блока.
5. **Неподключённый пин:**
   - Неподключенный внешний пин блока.
   - Неподключенный пин инстанса.
6. **Некорректная цепь:** Ошибки соединения (зависит от формата).
7. **Неподключённая цепь:** Цепь, не подключенная ни к пинам блока, ни к примитивам (может возникать при раскрытии иерархии).
